# 整体架构

| 层                      | 作用                  |
| ---------------------- | ------------------- |
| API            | 数据源                 |
| ✅ `.dx.jsonc`          | 👉 UI编排 + 资源映射（关键层） |
| View (React / dx view) | 实现具体 UI             |
| Contain Shell            | 渲染 & 生命周期管理         |

## 配置演进的细节

- 无配置阶段，整个portal前端需要在一个项目中
- gallery package阶段，将各个resource的前端项目分开，主动注册每一个项目的配置，但不负责UI编排
- .dx.jsonc，portal可以主动服务发现，将配置的控制权限分开，即更新/创建与读取的分离

微前端的“UI + 资源绑定配置层”
过去是 UI +逻辑写死在前端代码
现在是 JSON（结构） + View（实现）服务入口和基本元数据由配置控制

## 跨resource互相调用的能力
A_page -> B_page
A发起跳转请求
runtime收到并检查是否有B的能力
加载B的必须资源
正确加载


## runtime需要的核心能力
runtime ≈ 一个 runtime 微前端操作系统（OS）

extension = 应用
blade = 页面
dx.jsonc = UI schema
runtime = 内核

## 微前端需要的供给侧能力

- 提供自己所有能力MAP，类似于页面route，实际上是给runtime的注册信息
- 提供自己的默认能力展示，用于统一的资源显示
