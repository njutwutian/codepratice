# 整理关于闭包的知识点
## 书面定义

引用了另外一个函数作用域中变量的函数，通常是使用嵌套函数实现的。例如：

```javascript
function createSayFunction(who) {
    return function (what) {
        return `${who} says '${what}'`;
    }
}

const jackSay = createSayFunction("Jack");
const tomSay = createSayFunction("Tom");
jackSay("hello"); // Jack says 'hello'
tomSay("foo"); // Tom says 'foo'

```

## 关于函数作用域的一些背景知识，这是前情提要
在以下代码中
```js
function compare(a, b) {
    if (a < b) {
        return -1;
    } else if (a > b) {
        return 1;
    } else {
        return 0;
    }
}

let result = compare(5, 10)
```

这是函数内部的活动对象
```text
┌─────────────────────────────────────┐
│     compare() 的活动对象 (AO)        │
│  ┌─────────────────────────────┐    │
│  │  arguments: { 0: 5, 1: 10 } │    │  ← 伪数组，包含所有传入参数
│  │  a: 5                       │    │  ← 形参 a 被赋值为 5
│  │  b: 10                      │    │  ← 形参 b 被赋值为 10
│  │  (没有其他局部变量)          │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```
作用域链实际是一个包含指针的列表，如下图示
```text
┌─────────────────────────────────────┐
| 作用域链                             |
|-------------------------------------|
| 2. 全局变量对象，用于读取全局对象      |
|-------------------------------------|
| 1. 内部活动对象                      |
└─────────────────────────────────────┘
```
查找过程
```text
作用域链（从内到外查找）：

  第 1 位（最优先）  →  compare() 的活动对象 (AO)
                      包含: arguments, a, b

  第 2 位（次优先）  →  全局上下文的变量对象 (VO)
                      包含: this, result, compare, window/global...
```

闭包的作用域结构
```text
┌─────────────────────────────────────────────────────────────────┐
│  全局作用域 (生命周期: 永久)                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  createSayFunction: function 引用                      │   │
│  │  jackSay: ───────────────┐                            │   │
│  │  tomSay:  ───────────────┼──────────┐                 │   │
│  └───────────────────────────┼──────────┼─────────────────┘   │
│                              │          │                      │
│                              ▼          ▼                      │
│  ┌─────────────────────┐  ┌─────────────────────┐            │
│  │  闭包 1 (jackSay)    │  │  闭包 2 (tomSay)    │            │
│  │  [[Scope]] 指向:     │  │  [[Scope]] 指向:     │            │
│  │  ┌─────────────────┐ │  │  ┌─────────────────┐ │            │
│  │  │ who: "Jack"     │ │  │  │ who: "Tom"      │ │            │
│  │  └─────────────────┘ │  │  └─────────────────┘ │            │
│  │  生命周期: 直到       │  │  生命周期: 直到       │            │
│  │  jackSay = null      │  │  tomSay = null       │            │
│  └─────────────────────┘  └─────────────────────┘            │
│                              │          │                      │
│                              ▼          ▼                      │
│  jackSay("hello") 执行时:  tomSay("foo") 执行时:              │
│  ┌─────────────────────┐  ┌─────────────────────┐            │
│  │  当前 AO             │  │  当前 AO             │            │
│  │  what: "hello"       │  │  what: "foo"        │            │
│  │  作用域链:           │  │  作用域链:           │            │
│  │  [当前 AO] → [闭包1] │  │  [当前 AO] → [闭包2] │            │
│  │  输出: Jack says...  │  │  输出: Tom says...   │            │
│  └─────────────────────┘  └─────────────────────┘            │
│  执行完销毁当前 AO       执行完销毁当前 AO                    │
└─────────────────────────────────────────────────────────────────┘
```

## 究极理解
- 闭包在"设计理念"上，确实像极了 OO 中的"抽象类/工厂"；但在"设计模式"上，它更准确的定位是——"函数式编程"中的"高阶函数（函数工厂）"。
- 闭包出现内存泄漏本质上还是逻辑没有尽可能的函数化拆分，妄图借用直接使用对象的便利，如果严格遵循函数式编程的理念，其实没有问题
