# 下面是一些关于闭包的学习中的Q&A

Q: 为什么在函数的内部活动对象中会同时存在arguments和常规的a, b这种参数

A: 不是两份独立的数据，而是同一个值的两个不同"入口"。

`tip:` ES6+严格模式下两个不是同样的引用

---

Q: 为什么严格模式下是不同的引用
A: 根本原因：引擎为了实现的"命名映射"与"索引映射"同步关系，需要维护一个类似map的东西，将参数与arguments的索引关联
```text
┌─────────────────────────────────────────────────────────┐
│                 非严格模式                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌───────────────┐         ┌─────────────────┐        │
│  │ 形参存储区     │         │  arguments 存储  │        │
│  │ (Parameter)    │         │  (Arguments)    │        │
│  ├───────────────┤         ├─────────────────┤        │
│  │ value1 = 5    │◀─映射──▶│ [0] = 5         │        │
│  │ value2 = 10   │◀─映射──▶│ [1] = 10        │        │
│  └───────────────┘         └─────────────────┘        │
│                                                         │
│  ⚠️ 修改任一位置 → 引擎触发同步 → 更新另一个位置      │
│  ⚠️ 每次访问都可能触发"同步检查"                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
---
Q: 为什么箭头函数中没有arguments
A: 箭头函数的设计初衷是"轻量级函数"，用于：
- 简短的回调
- 不需要 this 绑定的场景
- 函数式编程风格

`tip:` 箭头函数是一个mini版的JS函数，它不需要照顾一些edge case，只需要完成输入输出的工作就行了
