# 2026-06-26 Agent-C 变更记录

## 当前目标

完成 SecretPalace 最小可用数据生产器：可生成、发送、失败记录、重放和报告输出。

补充完成 SecretPalace 全量 TypeScript 迁移：构建链路、源码文件、CLI 入口全部改为 TypeScript。

补充完成并发控制增强：默认值提升到 50，支持运行中方向键动态调整并发（上下 LogN、左右翻倍/减半），并支持配置上限。

补充完成 API v1 联调对齐：请求模型严格匹配 `POST /api/v1/events`，新增异常场景配置用于覆盖 `GM-404-SOURCE` 与 `GM-400`。

## 影响范围

- Node.js 模拟器子系统：`src/SecretPalace/**`
- 交接文档：`doc/changelog/development/2026-06-26-agentC.md`

## 变更清单

- 新增 CLI 入口：`src/SecretPalace/src/cli.js`
- 新增配置加载：`src/SecretPalace/src/config/loadConfig.js`
- 新增数据生成器：`src/SecretPalace/src/generator/*`
- 新增发送引擎：`src/SecretPalace/src/engine/*`
- 新增失败存储与重放：`src/SecretPalace/src/storage/failureStore.js`
- 新增指标与报告：`src/SecretPalace/src/report/*`
- 新增示例配置与模板：`src/SecretPalace/config/scenario.sample.json`、`src/SecretPalace/templates/event-template.json`
- 新增样例报告与说明：`src/SecretPalace/reports/sample-run-report.json`、`src/SecretPalace/README.md`
- TypeScript 迁移：新增 `tsconfig.json`、新增 `src/SecretPalace/src/**/*.ts`、删除 `src/SecretPalace/src/**/*.js`
- 构建脚本迁移：`package.json` 增加 `build`，`run/dry-run/replay` 改为编译后执行 `dist/cli.js`
- 并发控制增强：`run.concurrency` 默认 50、`run.ratePerSecond` 默认 50、新增 `run.maxConcurrency`（默认 1000）
- CLI 交互增强：运行中支持方向键调整并发，支持 `--no-interactive` 关闭交互
- 契约字段对齐：模板改为 `requestId/sourceCode/eventType/eventTime/payloadCount/payloadSummary`
- 错误码关联：HTTP 客户端解析统一响应体并提取 `code`，报告可区分 `GM-*` 失败原因
- 新增联调场景：`config/scenario.invalid-source.json`、`config/scenario.invalid-payload.json`

## 验证步骤

1. 进入目录：`src/SecretPalace`
2. 执行 dry-run：`node dist/cli.js run --config config/scenario.sample.json --dry-run`
3. 执行正常场景 run：`node dist/cli.js run --config config/scenario.sample.json --no-interactive`
4. 执行异常场景 run：`node dist/cli.js run --config config/scenario.invalid-source.json --no-interactive`
5. 执行异常场景 run：`node dist/cli.js run --config config/scenario.invalid-payload.json --no-interactive`
6. 执行 replay（可选）：`node dist/cli.js replay --config config/scenario.sample.json --input failures/failed-events.jsonl`
